# 양념장 B2B 사업 (AZAE 브랜드 — 라인 2)

박준호가 **"투자를 받기 위해 구조를 짜고 법인 전환을 하고 매출을 일으켜야 해서"** 진행 중인 핵심 사업. 2026-05-03 박준호 본인이 작성한 AZAE 위키에 따라 이 사업은 **AZAE 브랜드의 라인 2 (B2B 제조)** 로 명확히 정렬됨.

> AZAE 사업 체계 전체는 [[위키/주제/AZAE_브랜드_사업체계]] 참고.

## 1. 자기 진술 (원문)

> 투자를 받기 위해 구조를 짜고 법인 전환을 하고 매출을 일으켜야 해서, **양념장을 개발하고 이 양념장을 기반으로 B2B 사업을 진행**하려고 하고 있어. 그 일들이 잘 해결되어야 할 거라는 걱정이 있고.

---

## 2. 사업의 구조 (자기 진술 정리)

박준호가 풀어놓은 한 단락 안에 실은 **세 단계가 묶여있다**:

```
[양념장 개발] → [B2B 사업화] → [매출 발생] → [법인 전환] → [투자 유치]
```

각 단계가 서로 의존:
- **양념장이 있어야** B2B로 팔 수 있음.
- **매출이 있어야** 법인 전환의 의미가 있음.
- **법인이 되어야** 투자를 제대로 받을 수 있음.
- **투자가 들어와야** 다음 사업 (초콜릿/싸는음식/AI 에이전트) 의 운영 자본 확보.

→ 즉, 양념장은 **단순한 한 사업이 아니라 박준호 사업 포트폴리오 전체의 첫 도미노**.

## 2.5 B2B 영업 3채널 (AZAE 라인 3, 2026-05-04 신설)

박준호가 AZAE 위키에서 명시한 영업 채널 3개. 라인 2(제조) 가 만든 제품을 흘려보내는 통로.

### 채널 A — 프랜차이즈 본사 납품
- 대상: 외식 프랜차이즈 본사.
- 상품: 꼬막 소스 + 냉동 꼬막 **묶음**.
- 가치: 본사 입장에서 **신메뉴 개발 비용 절감**.

### 채널 B — 꼬막 공급업체 역납품 (역방향 공급망)
- 대상: 박준호에게 꼬막 원물을 공급하는 업체.
- 상품: AZAE 소스.
- 가치: 공급사가 자기 B2C 채널에서 박준호 소스를 **함께** 판매.
- 박준호 자기 표현: "**본인의 공급사가 본인 제품을 팔아주는 구조**".
- → 매타리(@matari.sool)의 Partners 협업 구조와 닮음 ([[위키/주제/벤치마크_호랑이굴_마타리]]).

### 채널 C — 케이터링 업체 납품
- 대상: 케이터링 회사.
- 가치: 행사 메뉴에 한식 시그니처 추가.

### 핵심 자세 (박준호 한 줄 정의)
> "**나는 B2B를 받지만, 그들은 B2C가 쉬워지는 비즈니스**."

→ 3 채널 모두의 공통 검증 기준. 페이지 [[위키/주제/AZAE_브랜드_사업체계]] 4.x 참고.

## 3. 식품가공기능사 자격증과의 정합성

박준호가 **식품가공기능사 필기 합격, 실기 예정** 인 이유가 여기서 명확해짐:

- 양념장 = 식품가공의 한 영역.
- 식품가공기능사 자격은 양념장 제조의 **현장 자격 / 영업 신고의 근거**.
- → 학과(호텔외식경영) + 자격증(식품가공) + 사업(양념장 B2B)이 한 줄로 정렬.
- 페이지 [[위키/자아/자격증_보유기술]] 의 "트랙 B" 가 이 사업과 직결됨.

## 4. 가게(아재꼬막)와의 관계 — R&D 연결

(추정) 양념장은 **아재꼬막에서 쓰는/개발한 양념장이 모태**일 가능성이 큼:

- 자영업자가 자기 매장에서 검증된 양념장을 상품화하는 것은 외식업의 정석 패턴.
- 아재꼬막 = **시제품 검증 환경**, 양념장 B2B = **사업 확장 단계**.
- → 가게가 무너지면 양념장의 정체성·맛·근거도 흔들림. 가게 운영의 중요도가 두 배.

## 5. 박준호의 걱정 (자기 진술)

> "그 일들이 **잘 해결되어야 할 거라는 걱정**이 있고."

이 한 줄 안에 박준호의 ENTJ Type Angst의 흔적이 있다.

- 박준호의 그림에서 양념장 B2B는 **반드시 성공해야 다음이 가능**. 따라서 실패의 무게가 큼.
- ENTJ J 의 함정 — "한 번에 안 되면 끝이다" 식의 압박감.
- 자기 점검 신호: 이 걱정이 추진력으로 쓰일 때는 좋은 자극, 자기 의심으로 변하면 [[위키/생각/안풀릴때_대처법_메타인지]] 의 행동 루틴(런닝·노래·관찰)으로 처리.
