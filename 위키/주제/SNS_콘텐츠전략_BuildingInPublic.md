# SNS 콘텐츠 전략 — Building in Public

박준호의 사업(AZAE)을 외부에 드러내는 채널 전략. 단순한 인스타 운영이 아니라 **B2B 리드 확보 + 신뢰 자산 축적** 을 위한 콘텐츠 시스템.

---

## 1. 운영 목표 (우선순위 순)

박준호 자기 정의:

| 우선순위 | 목표 |
|---|---|
| 1 | **B2B 리드 확보** — AI 컨설팅·SaaS·B2B 납품 잠재 고객 |
| 2 | **신뢰 자산 축적** — Building in Public 방식의 여정 공개 |
| 3 | **AZAE 브랜드 발견성 확보** — 콘텐츠 그래프상의 거점 |

→ 일반 인플루언서 모델("팔로워 수")이 아니라 **잠재 B2B 고객 발견 채널**.

---

## 2. 핵심 전략 결정

박준호가 명시적으로 결정한 항목:

| 항목 | 결정 |
|---|---|
| 계정 정체성 | 사장 실명 + AZAE 브랜드 결합 (예: `@junho.azae`) |
| 운영 방식 | **Building in Public** |
| 얼굴 노출 | 적극 (실명 + 얼굴 = 신뢰의 핵심) |
| 빈도 vs 밀도 | **밀도 우선** — 매일 못 올려도 한 편을 작품처럼 |

### 박준호의 핵심 통찰
> "**팔로워 100명도 다 잠재 B2B 고객.**"

→ 일반적 SNS 운영은 팔로워 수 게임. 박준호의 운영은 **잠재 고객 1명의 가치 게임**. ENTJ Crusader 의 "**가능한 큰 문제 해결**" 가치관 ([[위키/생각/일_가치관_문제해결_자유]]) 의 마케팅 버전.

## 3. Building in Public — 박준호식 구현

### 3.1 정의 (박준호 본인 표현)
사업 과정 자체를 실시간 공개하는 전략. 법인 전환·에이전트화·매출 결과·시행착오를 모두 보여줌.

### 3.2 박준호에게 적합한 이유
- **자기 자신이 이미 외부화 시스템 운영자** ([[위키/생각/나를_믿지않는다_다중메타인지]]). 위키도 그 시스템의 한 형태.
- **솔직함이 윤리 마지노선** ([[위키/생각/윤리_마지노선과_타협의정의]]). 거짓 없이 보여주는 것이 자기 가치와 정합.
- **실험실(가게) 모형** ([[위키/주제/AZAE_브랜드_사업체계]]) — 시행착오 자체가 콘텐츠 자산.

### 3.3 위험
- 너무 많이 공개하면 경쟁사가 시스템을 카피.
- 보호 장치: 박준호의 [[위키/생각/사람을_보는_눈]] 의 마지노선 — 기만 없이 공개하되, 일부 영업 비밀은 차후 컨설팅 상품의 부분으로 두고 비공개.

## 4. 콘텐츠 카테고리 (4갈래)

박준호 자기 비중 정의:

| 카테고리 | 비중 | 예시 |
|---|---|---|
| 🤖 에이전트 빌딩 일지 | 40% | "오늘 발주 자동화 만든 후기" + 화면 캡처 |
| 📊 운영 결과 공개 | 25% | "이번 달 인건비 ○○원 절감" |
| 🍳 가게·요리 일상 | 20% | 새 메뉴 구상, 요리 과정, 가게 풍경 |
| 🧠 시스템 인사이트 | 15% | "사장이 RACI 짜는 법" 등 |

### 분석
- **에이전트 빌딩 + 운영 결과 = 65%** — 즉, B2B 고객(자영업 사장) 의 가장 큰 관심사인 "AI로 진짜 뭘 할 수 있나" 에 집중.
- **가게·요리 일상 20%** — 신뢰의 인간적 베이스. 너무 많아지면 일반 푸드 인플루언서로 흐려짐.
- **시스템 인사이트 15%** — 박준호의 슈카월드식 톤 ([[위키/생각/슈카월드_콘텐츠톤_참조]]) 이 작동하는 영역.

→ 일반 자영업 SNS 와 가장 다른 점: **AI·시스템 콘텐츠가 65%**. 박준호의 사업이 단순 외식업이 아닌 시스템 빌더 사업임을 콘텐츠 비중으로 직접 표현.

## 5. 인스타그램 계정 구조 (박준호 설계)

### 5.1 Bio (확정 안)
```
AZAE | 사장의 푸드솔루션
한식 시그니처 + AI 운영 시스템
박준호 / 아재꼬막 사장
컨설팅·납품 문의 → DM
```

→ Bio가 곧 박준호의 마스터 한 줄 정의. 4 라인 사업 ([[위키/주제/AZAE_브랜드_사업체계]]) 이 4 줄로 압축.

### 5.2 하이라이트 5개 (매타리 차용)
| 하이라이트 | 내용 |
|---|---|
| **Order** | 가게 예약·주문 정보 |
| **B2B** | 소스·꼬막 납품 문의 |
| **Essence** | AZAE의 본질 (브랜드 스토리) |
| **With** | 협업 다이닝·프랜차이즈 케이스 |
| **AI Lab** | 에이전트 빌딩 일지 |

→ 하이라이트 = 사업 구조의 미니어처. 페이지 [[위키/주제/벤치마크_호랑이굴_마타리]] 의 마타리 모형 차용.

### 5.3 고정 핀 3개
1. **꼬막 결과물** (양념 + 무친 꼬막)
2. **꼬막 원물** (껍질 / 신선한 꼬막)
3. **양념·소스 만드는 과정** (사장 손)

→ 첫 진입자에게 "이 사람이 뭘 만드는지" 가 즉시 전달.
→ 매타리 모형: 결과 → 원료 → 과정. 박준호 적용도 동일 구조.

### 5.4 해시태그 전략
| 카테고리 | 태그 |
|---|---|
| 브랜딩 | `#아재꼬막` `#AZAE` |
| 카테고리 | `#꼬막` `#꼬막무침` `#한식맛집` |
| 이벤트/시리즈 | `#사장의에이전트` `#자영업AI일지` |
| 지역 | `#[가게 위치]맛집` |

## 7. 콘텐츠 그래프 시대 운영 원칙

박준호의 핵심 통찰 — 일반 SNS 사고와의 차이:

| 기존 (소셜 그래프) | 박준호가 가져갈 (콘텐츠 그래프) |
|---|---|
| 팔로워 수 / Bio 미관 | 개별 릴스의 도달·저장·검색 노출 |
| "이 계정 따라하기" | "왜 발견되는지" 메커니즘 분해 |
| 일관된 피드 그리드 | 첫 3초로 멈춰지는 후킹 |
| 팔로우 = 친구 소식 | 팔로워 100명도 다 잠재 B2B 고객 |

→ **알고리즘 시대의 SNS 운영을 ENTJ 시스템 빌더의 시각으로 풀어둔** 박준호식 정리. 박준호의 [[위키/자아/변신로봇_시스템빌더_징검다리]] 시스템 빌더 정체성이 SNS 영역에 적용.

## 8. 피드 레이아웃 전략

박준호 자기 결정:

> "**브랜드 색이 강조된 레이아웃 + 단일 필터 지속 사용**"

- 누룽지 브라운 + 쌀밥 크림 톤을 모든 사진에 동일 필터.
- 다크 사진과 라이트 미니멀을 교차 배치 (체크보드 패턴).

→ [[위키/자아/AZAE_디자인시스템]] 의 컬러 팔레트가 SNS 피드에서 직접 발현.

## 10. 위험 / 주의

### 10.1 Building in Public 의 정신적 무게
- 사업 과정 공개 = 매일 외부 시선에 자기 자신을 노출.
- ENTJ Type Angst("아무도 내 말 안 듣는다") + 박준호의 [[위키/생각/행복이_고장났다]] 가 결합되면, 외부 반응의 작은 부정도 크게 다가올 위험.
- 보호 장치: 박준호의 [[위키/생각/안풀릴때_대처법_메타인지]] 시스템.

### 10.2 콘텐츠 비중의 75:25 균형
- AI·시스템 콘텐츠 65% + 가게·요리 20% + 인사이트 15%.
- 가게·요리 비중이 너무 줄면 박준호의 정체성 핵 ("요리덕후") 이 흐려질 위험.
- 추적할 항목.

### 10.3 외식업 사장 정체성 vs AI 컨설턴트 정체성
- 박준호의 사업은 "외식업 사장이 AI 하는 것" 이지 "AI 컨설턴트가 외식업 하는 것" 이 아님.
- 콘텐츠 톤이 후자로 기울면 박준호의 차별점 (시대 진단의 "현장에서 직접 검증") 이 약해짐.
- 보호 장치: 가게·요리 일상 20% 비중을 지키는 것.
