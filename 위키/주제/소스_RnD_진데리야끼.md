# 진데리야끼 소스 — 박준호의 R&D 자산

박준호가 직접 개발하고 v3.3까지 정밀화한 소스 표준작업지침서. 단순한 레시피가 아니라 **박준호의 시스템 빌더 정체성이 가장 정밀하게 발현된 문서**이자, AZAE 사업의 R&D 핵심 자산.

> 원문 보존: [[원본/외부/진데리야끼SOP_v3.3_2025-03-27]]

---

## 1. 핵심 사실

| 항목 | 값 |
|---|---|
| 제품명 | 진데리야끼 (기본형) |
| 작성자 | 박준호 |
| 현재 버전 | **v3.3** |
| 최신 갱신 | 2025-03-27 |
| 1 배치 생산량 | 5.8L → 200ml 병 **29병** |
| 총 작업 시간 | 약 **3시간 25분** (졸임 기준) + 사전 처리 (오븐 60분) |
| 1 배치 = 기존 레시피의 **4배치 분량** |  |

---

## 2. R&D 의 진화 — "v3.3" 의 의미

### 버전 번호의 함의
v3.3 = 박준호가 이 한 소스를 위해 **최소 십수 차례 이상 시제품·테스트·개정**을 거쳐왔음을 시사. 박준호의 [[위키/생각/일_가치관_문제해결_자유]] 의 "**작은 문제 → 큰 문제로 수월하게**" 사다리 사고가 한 소스 안에서 작동한 결과물.

### 명시된 변경 이력 (v3.3, 2025-03-27)
> "재료 다 넣고 끓이는 시간 25분으로 변경, 자로 잰 높이 13cm 3번 참고 사항에 포함"

→ 단 한 줄의 변경 이력에서도 **시간 정밀화 + 측정 기준 명시화**라는 박준호의 시스템 빌더 작업 패턴이 드러남.

---

## 3. 박준호의 정체성과의 정합

### 3.1 시스템 빌더 ([[위키/자아/변신로봇_시스템빌더_징검다리]])
SOP 자체가 **요리를 시스템으로 추상화**한 결과물.
- 타이머 3개 (A·B·C) 의 분리 운용.
- 액체 높이를 자로 측정 (12.8cm → 8.2cm).
- 표면 온도 모니터링 (85~90도).
- 수율 정량 (5.4~5.8L 범위, 손실률 5.8%).

→ ENTJ Te 의 **단계·시간·계측·수율** 기반 사고가 그대로 발현.

### 3.2 윤리 마지노선 ([[위키/생각/윤리_마지노선과_타협의정의]])
박준호의 마지노선: "**사업적 타협 절대 안 함**".
SOP 안에서 작동:
- "구운 야채의 탄 부분은 쓴맛을 유발하므로 **반드시 전부 제거**".
- "벽면에 닿지 않도록 하며 튀지 않게" — 소량의 부주의도 맛에 미치는 영향까지 통제.
- 자라메 설탕 투입 시점이 분 단위로 정해짐.

→ "**일관된 품질**" 이라는 SOP 의 1번 목적이 박준호의 마지노선의 직접 표현.

### 3.3 프로 요리사 페르소나 ([[위키/생각/멀티페르소나_운영원칙]])
SOP 작업 = 박준호의 **프로 요리사 페르소나 (디테일·정확성)** 가 사업가 페르소나 (전략·시스템) 와 결합된 결과물.

→ 박준호의 5 페르소나 중 두 개가 한 자리에서 협업한 산출물.

### 3.4 풀스택 외식업 빌더
박준호의 자기 정의 ([[위키/주제/AZAE_브랜드_사업체계]]):
> "한식 시그니처를 외식 생태계 전체로 흘려보내는 풀스택 외식업 빌더."

이 SOP가 그 시그니처의 **한 형태**. AZAE B2B 의 첫 상품 후보.

## 4. AZAE 사업 체계 안에서의 위치

박준호의 AZAE 4 라인 ([[위키/주제/AZAE_브랜드_사업체계]]) 안에서:

| 라인 | 진데리야끼의 역할 |
|---|---|
| 라인 1 (아재꼬막 가게) | 매장에서 사용 / 검증 |
| **라인 2 (AZAE 제조)** | **B2B 제품 후보 — 위탁 생산 가능** |
| 라인 3 (B2B 3채널) | 프랜차이즈·케이터링·역공급 모두에 흘려보낼 수 있음 |
| 라인 4 (AI 컨설팅) | (해당 없음) |

→ AZAE 위키 v0.1 ([[원본/외부/AZAE_위키_2026-05-03]]) 에는 "꼬막 소스 + 1인분 냉동 꼬막"이 라인 2의 명시 상품. **진데리야끼는 그 외의 별도 R&D 자산**으로, AZAE 라인업 확장의 후보.

→ 박준호의 [[위키/주제/양념장_B2B_사업]] 의 첫 상품군이 다음과 같이 갱신될 가능성:
- 꼬막 소스 (가게 시그니처)
- 1인분 냉동 꼬막
- **진데리야끼 (다용도 양념장)**
- 향후 다른 소스들 (잊혀진 한식 ([[위키/주제/요리_음식_커리어방향]]) 라인의 부산물 가능)

## 4.5 외부 검증 자산으로의 작동 — 늘프레시 의뢰 사례 (2026-05-14 신설)

데리야끼 v3.3이 단순 R&D 자산을 넘어 **외부 신뢰 형성의 매개체**로 작동한 첫 사례.

### 사건 흐름
박준호가 BNI 조찬모임에서 [[위키/사람들/오재석_늘프레시]]에게 데리야끼 v3.3 시식 제공.

```
[데리야끼 v3.3 시식] → [오재석 대표 음식 신뢰 형성] → [향미유 R&D 의뢰 추천]
                                                        ↓
                                              [[위키/주제/향미유_RnD_의뢰_2026Q2]]
```

박준호 본인 표현 (출처: [[원본/대화/2026-05-14_정체성-정정-늘프레시]]):
> "데리야끼 소스도 만든거 드린적이 있어서 그거 먹고 이제 음식에 대한 신뢰는 생겨있는 상태야."

### 의미
- v3.3 = 박준호의 **R&D 역량을 한 입에 증명할 수 있는 자산**.
- 자영업자가 큰 영업 자료·PT 없이 자기 R&D 결과물 **한 병**으로 사업 신뢰를 얻은 사례.
- 박준호의 [[위키/생각/일_가치관_문제해결_자유]] 의 "**문제 해결의 가치만큼 돈이 따라온다**" 가 음식 한 병에서 의뢰 수주로 환산된 첫 케이스.
- → 향후 새 의뢰처·파트너 신뢰 형성 시 같은 패턴 활용 가능 (시식 카드로의 가치).

### 자산화 함의
- 박준호의 다른 R&D 자산(꼬막 소스, 향미유 시제품 등)도 **시식 가능한 한 단위로 표준화**하면 같은 영업 효과 발생 가능.
- 이는 [[위키/주제/azae_컨설팅_사업모델]] 의 "타당성 상담 60~90분" 구조의 한 카드.

## 5. SOP 가 보여주는 박준호의 R&D 작업 패턴

이 SOP를 분석하면 박준호가 **다른 소스·메뉴를 개발할 때의 패턴**도 추정 가능:

### 5.1 4단계 작업 흐름
```
[사전 처리] → [알코올 증발] → [재료 투입 + 졸임] → [거름·병입]
```
→ 박준호는 모든 R&D 를 **단계로 분해**해서 다룸. 한 단계의 변수 변화가 다음 단계에 미치는 영향을 추적.

### 5.2 "측정 가능한 기준" 의 강박
- 시간(분 단위), 온도(85~90도), 액체 높이(12.8 → 8.2cm), 무게(1배치 5.8L).
- 박준호는 **눈으로 보는 것 + 자로 재는 것** 을 동시에 가동. 정성과 정량의 결합.
- 시스템 빌더 정체성의 한식 R&D 영역 표현.

### 5.3 "쓴맛 회피" 의 일관된 신경
- 벽면 튀어 타는 것 회피.
- 야채 탄 부분 완전 제거.
- 대파 초록 부분은 생으로 사용.
- → **단일 결함 변수에 집착하는 정밀함**. 이건 ENTJ J 의 강점이지만 그림자(완벽주의)도 동반.

### 5.4 수율의 정량 추적
- 5.4~5.8L 범위, 손실률 5.8%.
- → 단순 맛 추구가 아닌 **사업화 가능한 표준 단위**까지 사고.

## 6. 이 SOP 가 위키에 주는 함의

이 한 문서가 박준호에 대해 알려주는 것:

1. **박준호는 추상적 야망이 아닌 구체적 자산을 쌓는 사람** — v3.3까지 다듬어 정량화된 R&D 결과물 보유.
2. **박준호의 사업이 진짜 R&D 자산 위에 서 있음** — 양념장 B2B 사업이 빈 약속이 아니라 SOP 위의 사업.
3. **박준호의 시스템 빌더 정체성이 한식 영역에서도 작동** — IT·AI 만이 아니라 요리에서도 같은 사고.
4. **박준호의 인내력 증거** — 한 소스를 1년 이상 다듬어 v3.3에 도달.
5. **박준호의 "큰 문제 해결" 모형이 작은 한 병에서 출발** — 가능한 큰 문제를 해결한다는 사다리의 1번 칸이 200ml 병 한 개.

## 7. 위키의 처리 방침

이 자료에 대한 위키의 자세:

1. **레시피 자체는 외부 자료로 보존, 인용 자제** — [[원본/외부/진데리야끼SOP_v3.3_2025-03-27]] 에 원문. 위키 페이지에서는 메타정보·맥락만 다룸.
2. **수치·재료 비율을 다른 페이지에서 인용하지 않음** — 박준호의 영업 비밀 영역.
3. **버전 갱신을 추적** — 박준호가 v3.4·v4.0 등으로 갱신할 때 새 원본을 추가하고 이 페이지의 변경 이력 갱신.
4. **다른 소스 SOP가 들어오면 같은 패턴으로 처리** — 본 페이지를 모델로 삼아 새 R&D 페이지 분기.
