# 메타인지 / 자기 이해 성향

자기 자신을 어떻게 다루려고 하는지에 대한 단서들. 사용자의 직접 진술이 아닌 **위키 시스템의 설계 자체**에서 추론된 것.

---

## 핵심 관찰

### 1. 자기 자신을 객체화해서 다룬다
- 자기 자신을 주제로 위키를 만든다는 발상 자체가 흔치 않음.
- 8개 카테고리(자아·사람·사건·시기·장소·주제·생각·질문)로 인생을 분류해서 보겠다는 시도.
- "사용자를 3인칭으로 지칭한다"는 톤 규칙을 직접 명시 — **자기 자신과 거리를 두고 관찰하려는 의지**.

### 2. 인식론적으로 신중하다
CLAUDE.md에서 직접 강조한 규칙들:
- "사실과 해석을 명확히 구분: '...라고 말함' vs '...로 보임' / '...로 추정됨'"
- "단정 짓지 않는다 — 모순·불확실은 명시"
- "비판적이지 않게, 관찰자 시점으로"

→ 사고할 때 **확실한 것과 불확실한 것을 분리**하려는 습관. 자기 정체성에 관해서도 마찬가지.

### 3. 시스템 사고를 한다
- 색인, 로그, 정비(lint), 고아 페이지 점검, 교차참조 — 이런 운영 개념을 위키 설계에 미리 포함.
- 즉, 시스템을 **만들 때부터 유지보수를 전제로** 설계.
- 단발성 일기가 아니라 장기 운영을 의도.

### 4. 자기 경계가 분명하다
- "민감한 내용이 들어올 때는 어디까지 적을지 사용자에게 확인"
- "절대 비공개로 두고 싶은 내용이 있다면 `.gitignore` 사용 검토"
- GitHub 저장소는 private.

→ 자기 노출의 **세분화된 경계**를 의식. 모든 것을 다 적지는 않겠다는 의도.

### 5. LLM과의 관계 설정이 성숙
- LLM을 "위키를 같이 쓰는 협업자"로 위치시킴.
- 단순 질의응답이 아니라 **운영 규칙·역할·톤까지 사전 정의**.
- 자기에게 가장 좋은 답을 끌어내기 위해 환경을 의도적으로 세팅.

## 추정되는 성격적 특징

(직접 진술은 아니지만 위 패턴에서 도출되는 가설들)

- 내성적·반성적 (introspective) 성향이 강할 가능성.
- 즉흥보다 **체계 선호**. 큰 그림을 미리 그리는 편.
- 한 번에 모든 것을 쏟아내기보다 **점진적 축적**을 신뢰.
- 감정보다 패턴·구조·관계에 먼저 주목하는 인지 스타일.
- "정확하게 기록되는 것"이 자기에게 중요한 가치일 가능성.

> 위 가설들은 인터뷰로 검증·수정 필요. 본인이 자신을 어떻게 보는지(자기상)와 시스템에서 드러나는 모습이 다를 수 있음.

## 위키 사용 의도 (추정)

가능한 해석들:

1. **자기 정체성 정리** — 흩어진 자기 데이터를 한 곳에 모으려는 단순한 동기.
2. **변화 추적** — 시간에 따라 변하는 자기 모습을 기록하고 비교하기 위함 (`갱신` 필드, `로그` 시스템이 단서).
3. **외부화된 기억** — 잊거나 왜곡되는 인간 기억을 보완하기 위해 외부 시스템으로 저장.
4. **사고 도구** — 단순 보관이 아니라, 적는 행위 자체로 사고를 명료화.
5. **LLM 파트너십 전제** — 미래에 자기에 대해 더 정확한 답을 받기 위해 LLM이 학습할 수 있는 베이스를 만드는 것.

→ 다섯 가지가 동시에 작동하고 있을 가능성이 가장 높음. **인터뷰에서 본인의 동기 직접 확인 필요**.
