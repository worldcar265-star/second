# 깃허브 동기화 사용법

이 보관소는 GitHub에 연결되어 있어 여러 컴퓨터에서 이어서 작업할 수 있다.
저장소: https://github.com/worldcar265-star/second

---

## 핵심 규칙 (한 줄 요약)

> **시작할 때 받아오고 (pull), 끝낼 때 올린다 (push).**

---

## 매일 쓰는 3박자

작업할 때마다 터미널을 열고 이 보관소 폴더에서 실행하면 된다.

### 1. 시작 — 최신 받아오기
```
git pull
```
다른 컴퓨터에서 작업한 게 있으면 가져온다. 새 컴퓨터든 같은 컴퓨터든 작업 시작 전에 항상 한 번.

### 2. 작업 — Obsidian에서 자유롭게 노트 작성/수정/삭제

### 3. 마무리 — 변화를 GitHub에 올리기
```
git add -A
git commit -m "오늘 한 일 메모"
git push
```
- `git add -A` : 바뀐 모든 것을 다음 저장에 포함
- `git commit -m "..."` : 메모와 함께 도장 찍기
- `git push` : GitHub로 올리기

---

## 새 컴퓨터에서 처음 시작할 때

1. 터미널 열기 (Cmd+Space → "터미널" → 엔터)
2. 받고 싶은 위치로 이동 (예: `cd ~/Documents`)
3. 보관소 받기:
   ```
   git clone https://github.com/worldcar265-star/second.git
   ```
4. GitHub 인증 (한 번만):
   ```
   gh auth login
   ```
5. 이름·이메일 등록 (한 번만):
   ```
   git config --global user.name "worldcar265-star"
   git config --global user.email "278569478+worldcar265-star@users.noreply.github.com"
   ```
6. Obsidian 열고 → "다른 보관소 열기" → 받은 폴더 선택 → 끝

---

## 자주 쓰는 점검 명령

| 명령 | 무엇을 보여주는지 |
|---|---|
| `git status` | 지금 어떤 변화가 있는지 |
| `git log --oneline -10` | 최근 도장(커밋) 10개 보기 |
| `git pull` | 본사 최신본 받아오기 |

---

## 주의

- 여러 컴퓨터에서 **같은 파일을 동시에** 편집한 뒤 양쪽에서 push 하면 충돌이 생긴다.
- 충돌을 피하려면: **시작 전 `git pull`, 끝낸 후 `git push`** 만 지키면 된다.
- 충돌이 나도 데이터가 사라지진 않는다. 막히면 Claude한테 물어보자.
