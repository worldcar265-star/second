# 세컨드 브레인 — "나 자신" 위키

이 저장소는 사용자(@worldcar265-star) 자신에 대한 살아있는 위키이다. LLM 위키 패턴(LLM이 원본을 읽고 점진적으로 위키 페이지를 만들고 유지하는 방식)을 따른다.

## 위키의 주제

**나 자신.** 외부 지식이 아니라, 사용자가 누구이고, 무엇을 경험했고, 어떤 생각을 하는지.

이 위키의 모든 페이지는 사용자에 대한 이해를 쌓는 데 기여해야 한다.

## 폴더 구조

### `원본/` — 사용자가 직접 입력한 원본 자료. **수정·삭제 금지. 읽기만.**
- `일기/` — 날짜별 자유 기록 (예: `2026-04-30.md`)
- `대화/` — Claude와의 의미있는 대화 정리본
- `미디어/` — 사진·음성·영상 등
- `외부/` — 외부에서 가져온 자료 중 자기와 관련된 것 (인용, 기사 등)

하위 폴더는 처음부터 다 만들 필요 없다. 첫 자료가 들어올 때 해당 폴더가 생긴다.

### `위키/` — LLM이 작성·유지하는 페이지. 사용자는 읽고 LLM이 쓴다.
처음부터 모든 카테고리를 만들지 않는다. 입수 과정에서 필요한 것부터 자라게 한다. 통상 다음을 사용:

- `자아/` — 정체성, 핵심 가치관, 자기 묘사
- `사람들/` — 관계 인물 (한 사람당 한 페이지)
- `사건/` — 의미있는 경험·사건
- `시기/` — 인생의 시기 (예: 학창시절, 2024년 가을)
- `장소/` — 의미있는 장소
- `주제/` — 반복되는 패턴·테마 (예: 직업불안, 창작욕)
- `생각/` — 신념, 의견, 변화하는 견해
- `질문/` — 미해결 질문, 탐구 거리

새 카테고리가 필요해지면 사용자와 합의 후 추가.

### 색인·로그
- `색인.md` — 모든 위키 페이지 카탈로그. 매 입수마다 갱신.
- `로그.md` — 시간순 작업 기록. 매 입수·질문·정비마다 항목 추가.

## 운영 워크플로우

### 입수 (Ingest)

사용자가 원본을 추가하거나 자기 이야기를 들려주면:

1. 핵심 내용을 짧게 짚어본다 (사용자와 확인)
2. 원본을 `원본/<적절한 하위 폴더>/`에 저장 (이미 사용자가 저장했다면 위치 확인만)
3. 영향받을 위키 페이지 식별 — 새로 등장한 인물·사건·주제·장소·시기·생각이 있으면 페이지 생성
4. 기존 페이지에 정보 추가·갱신, [[교차참조]] 보강
5. `색인.md` 갱신 — 새 페이지 추가, 기존 항목 한 줄 요약 갱신
6. `로그.md`에 입수 항목 추가
7. 사용자에게 어떤 페이지가 어떻게 바뀌었는지 짧게 보고

한 번의 입수가 위키 페이지 5~15개를 건드리는 건 정상이다. 부지런히 연결하고, 빠뜨리지 않는다.

#### 페르소나 대화 입수 (특수 케이스)

사용자가 핸드폰/맥미니 Claude Project (페르소나 에이전트) 와 나눈 대화 결과를 **입수 패키지** 형식으로 제출하면 (`원본 보존판` + `위키 페이지 후보` + `색인·로그 갱신안` + `보강 권장`), 위 1~7 단계를 그 패키지 내용에 따라 수행한다. 입수 패키지의 형식·작성 원칙은 [[위키/자아/_AI에게_나를_소개하기]] 참고. 이때 *원본 보존판*은 `원본/대화/YYYY-MM-DD_[주제].md` 로 저장 (대화 카테고리 — 인터뷰와 동일 처리).

### 질문 (Query)

사용자가 자기 자신에 대해 물으면:

1. `색인.md`에서 관련 페이지 찾기
2. 해당 페이지들 읽고 종합
3. 답변 시 `[[페이지명]]` 인용
4. 답변에서 새로운 통찰·정리·종합이 나왔다면, 위키에 별도 페이지로 보존할지 사용자에게 묻기 (대화에서 사라지지 않게)
5. `로그.md`에 질문 항목 추가

### 정비 (Lint)

사용자가 "정비해줘", "위키 점검", "건강검진" 류로 요청하면:

- 페이지 간 모순되는 진술 찾기
- 오래된·갱신 안된 정보 찾기
- 고아 페이지 (들어오는 링크 없음) 찾기
- 자주 언급되지만 페이지 없는 개념 찾기
- 빠진 교차참조 보강
- 다음에 사용자가 탐구하면 좋을 질문·자료 제안
- `로그.md`에 정비 항목 추가

## 페이지 작성 규칙

### Frontmatter
모든 위키 페이지는 다음을 포함:

```yaml
---
유형: 자아 | 사람 | 사건 | 시기 | 장소 | 주제 | 생각 | 질문
생성: YYYY-MM-DD
갱신: YYYY-MM-DD
원본: [[원본/일기/2026-04-30]]
태그: [...]
---
```

### 본문 구조
- **첫 줄(또는 첫 단락)**: 한 줄 요약 — 이 페이지가 무엇에 대한 것인지
- **본문**: 자유 형식. 사실 위주의 섹션과 해석 위주의 섹션을 분리하면 좋음
- **링크**: 다른 페이지 참조는 `[[경로/페이지명]]` 또는 `[[페이지명]]`
- **인용**: 원본 인용 시 출처 명시
  > 원문 (출처: [[원본/일기/2026-04-30]])

### 톤
- 한글로 작성
- 사용자를 3인칭으로 지칭한다 ("사용자는...", 사용자가 이름·호칭을 정해주면 그것으로)
- 사실과 해석을 명확히 구분: "...라고 말함" vs "...로 보임" / "...로 추정됨"
- 단정 짓지 않는다 — 모순·불확실은 명시
- 비판적이지 않게, 관찰자 시점으로
- 민감한 내용(타인의 비밀, 미공개 사실)이 들어올 때는 사용자에게 어디까지 위키에 적을지 확인

## 색인 형식 (`색인.md`)

카테고리별로 묶고, 각 페이지는 한 줄:
- `[[페이지명]] — 한 줄 요약 (원본 N개)`

비어있는 카테고리는 `*아직 페이지 없음*` 으로 표시.

## 로그 형식 (`로그.md`)

각 항목은 `## ` 헤더로 시작:
- `## [YYYY-MM-DD] 입수 | 제목`
- `## [YYYY-MM-DD] 질문 | 질문 요약`
- `## [YYYY-MM-DD] 정비 | 작업 내용`

본문은 짧게. 영향받은 페이지 목록 포함. 가장 최근 항목이 맨 아래.

다음 항목들은 사용자가 직접 "패스/제외"로 표시. 인터뷰·자료 분석 시 다루지 않음.

- **가족 (부모·형제자매)** — 2026-05-01 인터뷰에서 패스. 위키에 페이지 만들지 않음. 다른 페이지에서도 언급 자제.
  - 단, [[위키/사람들/후추_고양이]]는 본인이 "가족" 항목에 직접 적은 존재이므로 유지.
- **출생지 (시·구 단위)** — 1차 인터뷰에서 패스. "정신적 고향=광주"는 본인 진술이므로 사용 가능.

## 인터뷰 운영 원칙 — 현재 우선, 과거는 필요시

2026-05-01에 사용자가 명시한 운영 방향:

> 지금 내 생각·상태부터 위키에 채워 넣고, 과거 정보들은 나중에 필요할 때 그때그때 깊이 파서 보강한다. 처음부터 시간 순으로 모든 이력을 다 캐 들어가지 않는다.

이를 따르는 인터뷰·페이지 작성 규칙:

1. **새 주제에 들어갈 때 첫 질문은 "지금 어떤가"·"지금 무슨 생각인가"** 부터. 시간 순 이력은 묻지 않는다.
2. **과거는 필요할 때만** — 현재의 어떤 항목을 이해하기 위해 과거 사건이 결정적이라면, 그때 본인에게 짧게 물어 해당 부분만 보강한다. 통째로 이력을 캐는 인터뷰는 하지 않는다.
3. **"지금 중심" 페이지를 우선 생성** — 각 영역(일·관심사·가치관·일상 등)은 "현재 상태" 페이지를 먼저 만들고, 과거 사건/시기 페이지는 본인이 원할 때 추가 분기.
4. **이력서식 항목은 사용자가 명시적으로 요청할 때만** — "내 일자리 이력 다 정리해줘" 같은 요청이 명시되기 전까지는 시도하지 않는다.

이 원칙은 시기/사건 카테고리에도 적용 — 시기 페이지는 "그 시기에 살았던 흔적"이 자연스럽게 다른 페이지에서 언급될 때, 그제서야 페이지로 분기.
